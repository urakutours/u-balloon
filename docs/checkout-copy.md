> 本ファイルは SiteSettings 管理画面・チェックアウト各ブロック・ギフト系マスタデータの日本語文言を一括管理する仕様書です。
> 実装者・デザイナーはこのファイルをそのままコピペで使用できます。
>
> ブランドトーン: やさしい・祝祭感・カジュアルすぎず、フォーマルすぎない。
> 命名原則: 注文者 / 送り先 / ラッピング / メッセージカード で統一（Section 5 参照）。
> 旧称「送り主」は 2026-04-20 に「注文者」へリネーム（内部 DB カラム名 sender/senderName 等は互換性のため維持）。

## Section 5. 用語統一マッピング表

迷ったらこの表に従ってください。UI・コード・ドキュメント・メールテンプレート全てで統一します。

| 用途 | 採用する表記（統一後） | 廃止する表記（旧） |
|---|---|---|
| 商品の到着希望日 | **到着希望日** | 配送希望日 / 配達希望日 / お届け希望日 |
| 商品を届ける先 | **送り先** | 配送先住所 / お届け先 / お届け先住所 / 配送先 |
| 飾るイベントの名前 | **イベント名** | 用途 / 使い道 / ご利用シーン |
| バルーンを使う日 | **使用日** | イベント日時 / 使用予定日 / 飾り付け日 |
| 注文者本人 | **注文者** | 送り主 / お客様情報 / ご注文者 / 購入者 |

### 補足ルール
- 「注文者」「送り先」はチェックアウトおよびメール本文で統一する。
- 管理画面のコレクション名（PayloadCMS）の内部フィールド名は互換性のため `sender` / `recipient` のまま維持。ラベル・UI 表示のみ「注文者」に統一する。
- 「お客様」は呼びかけ（「お客様のご登録情報」など）に限って使用し、フィールド名には使わない。
- 「使用日」と「到着希望日」は意味が異なるため、UI 上で必ず並列に表示せず、セクションを分けて誤認を防ぐ。
- 英語キー（コード側）対応:
  - 到着希望日 → `deliveryDate`（`desiredArrivalDate` からリネーム検討）
  - 送り先 → `recipient`
  - 注文者 → `sender`（内部フィールド名は維持、表示のみ「注文者」）
  - イベント名 → `eventName`
  - 使用日 → `usageDate`
  - 使用時間 → `usageStartTime`
