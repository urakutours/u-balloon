## Step 3: エラーハンドリングとエッジケース

以下のケースでダッシュボードがクラッシュしないか確認してください。

### テストケース

```
1. 注文データが0件の状態でダッシュボードを開く
2. ユーザーデータが0件の状態でダッシュボードを開く
3. 期間タブを高速で連打する（APIリクエストの競合）
4. カスタム期間で未来の日付を選択する
5. カスタム期間で開始日 > 終了日を設定する
6. ネットワークエラー時（APIが500を返す場合）
7. GA4設定が不完全な状態（測定IDはあるがプロパティIDが無い）
```

### 対応方針

- データが0件 → 各コンポーネントに適切なプレースホルダーを表示
- API競合 → 最後のリクエストの結果のみ反映（前のリクエストは破棄）
  ```typescript
  // AbortController を使うパターン
  useEffect(() => {
    const controller = new AbortController();
    fetchStats(period, { signal: controller.signal });
    return () => controller.abort();
  }, [period]);
  ```
- 不正な期間指定 → バリデーションでブロック、またはAPIが400を返す
- ネットワークエラー → 「データを読み込めませんでした」メッセージ + リトライボタン

## Step 5: パフォーマンス確認

### API レスポンス速度

```
[ ] /api/dashboard/stats のレスポンスタイムを確認
    目標: 500ms以下（注文数が少ない現段階では余裕があるはず）

  キャッシュ（60秒程度のインメモリキャッシュ）を検討

## カスタムダッシュボード

### 概要
Payload CMS のデフォルトダッシュボードを、EC運営に最適化したカスタムダッシュボードに置き換えています。

### ファイル構成
- `src/components/Dashboard/` — ダッシュボードコンポーネント群
- `src/endpoints/dashboard.ts` — KPI集計エンドポイント
- `src/lib/ga4.ts` — GA4 Data API連携

### GA4 連携の設定手順
1. Google Cloud Console でプロジェクトを作成
2. 「Google Analytics Data API」を有効化
3. サービスアカウントを作成し、JSONキーをダウンロード
4. GA4 管理画面でサービスアカウントに「閲覧者」権限を付与
5. 環境変数に設定:
   ```
   GA4_SERVICE_ACCOUNT_KEY=<JSONキーの内容>
   ```
6. Payload 管理画面 > サイト設定 > GA4プロパティID に数値を入力

### 環境変数
| 変数名 | 必須 | 説明 |
|--------|------|------|
| GA4_SERVICE_ACCOUNT_KEY | いいえ | GA4 Data API 用のサービスアカウントJSONキー |
```
